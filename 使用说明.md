# AeroIntel Daily 使用说明

这是你的航空科技主题资讯看板项目。你先不用理解所有代码，先记住：网页负责展示，`data/news.json` 负责存新闻，`feeds.json` 负责配置新闻源。

## 1. 怎么打开网页

最简单的方法：

1. 双击 `start-server.bat`
2. 不要关闭弹出的黑色窗口
3. 打开浏览器，访问 `http://127.0.0.1:4173`

如果打不开，先看黑色窗口里有没有这句话：

```text
AeroIntel Daily running at http://127.0.0.1:4173
```

如果没有这句话，说明网站服务没有启动成功。

## 2. 为什么 `http://127.0.0.1:4173` 打不开

`127.0.0.1` 不是互联网网址，它只代表“你自己这台电脑”。所以它能打开的前提是：你这台电脑上已经运行了网站服务。

常见原因：

- 没有双击 `start-server.bat`
- 双击后黑色窗口马上关掉了
- 黑色窗口被你关闭了
- 电脑找不到 Node.js
- 4173 端口被其他程序占用
- 你在手机或另一台电脑上打开了这个地址

正确做法：

1. 先双击 `E:\webmaking\start-server.bat`
2. 保持黑色窗口打开
3. 再用同一台电脑的浏览器打开 `http://127.0.0.1:4173`

如果你想停止网站服务，在黑色窗口里按 `Ctrl + C`，然后输入 `Y` 回车。

## 3. 每个文件是做什么的

- `index.html`：网页结构，决定页面上有哪些区域和文字。
- `styles.css`：网页外观，决定颜色、布局、航空科技风格。
- `app.js`：网页功能，负责读取新闻数据、搜索、筛选、分类展示。
- `data/news.json`：新闻数据，网页会从这里读取资讯。
- `feeds.json`：新闻源配置，后续添加 RSS 地址就在这里改。
- `scripts/update_news.mjs`：更新真实新闻的脚本。
- `start-server.bat`：一键启动本地网站。
- `update-news.bat`：一键抓取真实新闻。
- `.github/workflows/update-news.yml`：以后上传到 GitHub 后，用来每天自动更新新闻。

## 4. 怎么修改页面文字

打开 `index.html`，找到你想改的中文文字，直接修改后保存。

比如网站名在这里：

```html
<strong>AeroIntel Daily</strong>
```

你可以改成：

```html
<strong>我的航空科技日报</strong>
```

保存后刷新浏览器即可。

## 5. 怎么手动改新闻内容

打开 `data/news.json`。

每一条新闻大概长这样：

```json
{
  "id": "aero-001",
  "title": "新闻标题",
  "category": "aerospace",
  "region": "domestic",
  "source": "来源",
  "publishedAt": "2026-05-12T08:00:00+08:00",
  "priority": "high",
  "summary": "一句话摘要",
  "tags": ["卫星", "火箭"],
  "url": "https://example.com"
}
```

常用分类：

- `aerospace`：航空航天
- `automation`：自动化
- `electronics`：电子信息
- `computing`：计算机
- `finance`：金融
- `security`：战争安全

常用地区：

- `domestic`：国内
- `global`：国外

## 6. 怎么更新真实新闻

真实新闻更新分 3 步：

1. 找 RSS 新闻源
2. 把 RSS 地址填进 `feeds.json`
3. 双击 `update-news.bat` 更新新闻

### 6.1 什么是 RSS

RSS 可以理解成“新闻网站提供给程序读取的新闻列表”。你的网页不会直接去网页上乱抓内容，而是优先读取 RSS，这样更稳定，也更尊重原网站。

RSS 地址一般长这样：

```text
https://www.nasa.gov/rss/dyn/breaking_news.rss
https://spacenews.com/feed/
https://www.technologyreview.com/feed/
```

### 6.2 怎么找 RSS 地址

可以用这些方法：

- 在新闻网站底部找 `RSS`、`Feed`、`订阅`
- 在搜索引擎搜：`网站名 RSS`
- 打开网站后试试常见地址：`/feed/`、`/rss/`

比如：

- NASA：`https://www.nasa.gov/rss/dyn/breaking_news.rss`
- SpaceNews：`https://spacenews.com/feed/`
- MIT Technology Review：`https://www.technologyreview.com/feed/`
- IEEE Spectrum：`https://spectrum.ieee.org/rss/fulltext`

不是所有网站都有 RSS。如果没有 RSS，后面可以再做“新闻搜索 API”或“网页抓取版”，但第一版建议先用 RSS。

### 6.3 怎么添加一个新闻源

打开 `feeds.json`，你会看到类似内容：

```json
[
  {
    "name": "NASA Breaking News",
    "url": "https://www.nasa.gov/rss/dyn/breaking_news.rss",
    "category": "aerospace",
    "region": "global",
    "tags": ["NASA", "航天"]
  }
]
```

每一项的意思：

- `name`：新闻源名字，自己取，页面里会显示。
- `url`：RSS 地址，必须是真实可访问的 RSS。
- `category`：归类到哪个板块。
- `region`：国内还是国外。
- `tags`：默认标签，可以写多个。

如果你要新增一个新闻源，在上一项后面加逗号，再加一段新的：

```json
{
  "name": "SpaceNews",
  "url": "https://spacenews.com/feed/",
  "category": "aerospace",
  "region": "global",
  "tags": ["商业航天", "卫星"]
}
```

注意：JSON 格式很严格。每一项之间要有英文逗号，最后一项后面不要加逗号。

### 6.4 怎么执行更新

最简单的方法：

1. 双击 `update-news.bat`
2. 等黑色窗口显示 `Wrote xx items`
3. 如果网站已经打开，刷新浏览器

命令行方法：

```powershell
cd /d E:\webmaking
node scripts/update_news.mjs
```

更新成功后，新的新闻会写入：

```text
E:\webmaking\data\news.json
```

### 6.5 如果更新失败怎么办

常见提示：

```text
WARN 某某新闻源: HTTP 403
```

意思是这个网站不允许程序读取，可以先换一个 RSS。

```text
WARN 某某新闻源: fetch failed
```

可能是网络问题、地址写错、网站临时打不开。

```text
JSON.parse error
```

大概率是你改 `feeds.json` 时少了逗号、引号，或者最后多了逗号。

## 7. 后续怎么变成真正的网址

后面可以把整个 `E:\webmaking` 上传到 GitHub，然后开通 GitHub Pages。

这样你会得到一个可以访问的网址，而且 `.github/workflows/update-news.yml` 可以每天自动更新新闻数据。
